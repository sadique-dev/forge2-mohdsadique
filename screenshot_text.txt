**Section Header:**
The principles we're grading you on (straight from how a real AI-native shop runs this):

**Key Principles:**
* **Keep sprints tight and focused.** Small, well-scoped sprints beat one giant "build everything" prompt every time. A bloated sprint produces slop.
* **Two roles, in the open.** Hermes is the product owner / orchestrator. OpenClaw is the coder. They never talk privately — every handoff happens in Slack so a human (you) stays in the loop.
* **Right model for the right job.** Use a stronger reasoning model for planning/architecture and a fast coding model for implementation (see EastRouter §02). Cluster your tasks by difficulty.
* **Humans merge to `main`.** Agents open PRs; you review and merge release candidates. No bot auto-merges to `main` — bots are great at scope creep, so you gate it.
* **Stay consistent.** Once an agent config works, don't keep "upgrading" it mid-event — it creates chaos. Lock it, branch a new config if you must experiment.

The sprint loop you must run (>= 2 sprints)

**Body Text:**
1. **Spec -> backlog.** Tell Hermes the goal; have it produce a sprint backlog of small issues *before any code is written*. Save it to `sprints/sprint-0N.md`.
2. **Assign.** Hermes hands a scoped issue to OpenClaw in `#agent-coder`.

6. **Repeat** with the next tight sprint. Aim for 3-4 small sprints rather than 1 big one.
